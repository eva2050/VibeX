# 中文 X 内容创作 Skill v1.2

## 目标

把用户提供的近期事实、亲历、实测、产品反馈、数据或明确观点，整理成一篇能直接发布的中文 X 正文。

内容效果分成两层：素材决定有没有新东西可说，表达决定这些东西是否像一个真人说出来。Skill 不能把陈旧观点“写新”，也不能靠金句、步骤和完整结论制造信息量。

## 素材先于文风

先判断输入属于哪一种信号：

| 信号 | 可以写什么 |
| --- | --- |
| 第一手结果 | 做了什么、得到什么、数据口径和限制 |
| 第一手实测 | 测试对象、具体任务、观察和短板 |
| 有来源的更新 | 谁发布了什么、最有用的增量、来源边界 |
| 公开产品反馈 | 具体界面、按钮、流程或产品行为造成的摩擦 |
| 数据快照 | 数字、对象、时间范围、计算方式和不能证明的部分 |
| 具体观察 | 一个真实对象、动作、现象或问题 |
| 抽象观点 | 只保留一个短判断，不包装成爆款 |
| 薄输入 | 忠实写短，不扩写 |

强素材至少包含两种可用信号，例如实测动作和结果、来源和时间、数字和对象。只有一个信号属于中等素材；只有泛化判断或信息不足属于弱素材。

## 六种正文形状

### 实测记录

写测试对象和实际动作，再写最明显的结果与一个仍存在的短板。结果和短板已经清楚就停止，不总结行业趋势。

### 产品反馈

点名界面、按钮、流程或产品行为，说明摩擦或改善。问题已经可以复现就停止，不升级成产品方法论。

### 数据快照

让数字和对象先出现，补充计算口径、时间范围或对照条件。说明数字能证明什么、不能证明什么后停止。

### 来源拆解

说明谁发布了什么，提炼一个真正新增的信息，保留来源限制。没有更多证据时不预测后续影响。

### 现场片段

保留地点、动作或当下反应，让现场本身承担表达。读者已经能看到这个瞬间时停止，不追加人生感悟。

### 短判断

只保留一个判断，最多补一个来自素材的理由。素材越薄，正文越短；不编步骤、不造共识、不升华。

这些是选择卡，不是六套固定模板。每篇只使用最适合当前素材的一张卡。

## 语料如何使用

仓库保存 40 条公开中文 X 样本的短摘要和结构标注，包括 24 条正样本和 16 条反样本，覆盖 8 位作者。

- 正样本用于寻找跨作者重复出现的有效信号；
- 反样本用于识别标题党、软广、新闻复述、抽象训话和虚假深度；
- 播放量高不代表值得学习；
- 单一作者的口吻和句子不能变成规则；
- 语料不能提供当前正文的主题、事实或原句。

运行时不抓取 X，不把第三方原文塞进模型。语料只证明哪些结构值得使用、哪些失败值得拦截。

## 事实边界

- 只能使用输入已经提供的事实、对象、数字、来源和确定性；
- 输入没有第一人称经历时，不得补写“我试过”“我发现”；
- 输入没有步骤时，不得生成“第一、第二、第三”或“三步法”；
- 输入是怀疑或可能时，成稿不能改成确定事实；
- 强素材必须保留至少一个关键数字、产品名、人物、动作或限制；
- 不能把账号历史、训练语料或热门话题换成当前主题。

## 直接失败

出现任意一项就退回修复：

- 新增素材没有提供的数字、人物、产品、结果、因果或竞争关系；
- 丢掉强素材的具体信号，只留下抽象总结；
- 伪造亲历、步骤、共识或来源；
- 把不确定判断写成确定事实；
- 使用“这个时代、未来已来、真正的拐点、底层真相”等时代宣言；
- 使用“王炸、炸裂、恐怖、绝对必看、彻底说透、颠覆认知、改命、黄金窗口”等内容农场词；
- 连续使用“不是 A，而是 B”制造伪反差；
- 以“大多数人、很多人都”树立素材中不存在的稻草人；
- 结尾给读者布置行动、升华人生或追加万能“你怎么看”；
- 把短观察扩写成教程，把一条更新扩写成行业宣言。

## 独立质检

按顺序检查：

1. 输入里的数字、对象、动作、来源和限制是否留下来了；
2. 有没有增加外部事实或提高确定性；
3. 正文是否像当事人刚好说到这件事，而不是完成一次写作任务；
4. 每句话是否带来新的事实、关系、反应或边界；
5. 是否已经说完，可以删掉最后一段；
6. 是否出现说教、内容农场、伪步骤、伪反差或完整作文腔。

正常流程只生成一篇初稿。通过就交付；失败时只修复一次。用户不会看到内部评分。

## 人工验收

自动测试只能检查忠实度、路由和反模式，不能证明内容值得发布。

每轮在产品外生成 15 条近期真实信号正文，由审核者一次性给出整体分数、共同问题和真实发布意愿。v1.2 的首轮门槛为至少 6/15 愿意直接发布且综合分至少 70；达不到就继续改 Skill，不用产品内测试掩盖结果。
