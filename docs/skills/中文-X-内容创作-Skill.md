# 中文 X 内容创作 Skill v1.1

## 目标

把用户提供的想法、经历、产品观察或文章片段，整理成一篇可以直接发布的中文 X 正文。

衡量标准不是“像一篇写得很完整的作文”，而是：真实、具体、有信息、像本人自然说话。粗糙但有内容，优先于精致但空洞。

## 输入契约

输入至少需要包含一个可写信号：

- 具体动作：试了、删了、发布了、失败了、换了什么；
- 具体对象：产品、工具、功能、用户或工作流；
- 具体结果：数字、变化、卡点、反馈或下一步；
- 有边界的观点：我觉得、可能、未必、正在发生什么。

Skill 只能使用输入已经提供的事实。输入没有第一人称经历时，不得补写“我试过”“我发现”；输入没有数字、人物或结果时，不得制造这些细节。

## 一篇正文的形状

1. 先放具体信号：事件、动作、数字、对话、产品行为或过程。
2. 说明这个信号为什么值得注意，必要时补充动作之间的关系。
3. 最后写有边界的个人判断，让观点从前面的事实自然长出来。

如果素材很薄，就写短。不得用宏大结论、对仗金句或空泛总结伪造深度。

## 六类内容

| 类型 | 推荐结构 |
| --- | --- |
| 产品观察 | 产品行为或使用断点 → 观察 → 判断 |
| 工具体验 | 我做了什么 → 遇到什么 → 为什么留下或放弃 |
| Build in public | 这次改了什么 → 结果 → 当前取舍 |
| 失败复盘 | 发生了什么 → 哪里判断错了 → 现在如何理解 |
| 行业观点 | 具体变化或约束 → 有边界的个人判断 |
| 工作流框架 | 保留原步骤 → 用动作说明 → 不增加步骤或承诺 |

## 语气规则

- 写“我怎么做、我看到了什么”，少写“你应该怎么做”。
- 没有第一人称素材时，直接写观察，不伪装成亲历者。
- 可以口语化，可以保留短句和停顿，不强求三段式金句。
- 每一段都必须增加信息，不能只换一种方式重复上一段。
- 允许长内容，但长度来自事实、过程和推理，不来自重复。
- 不模仿某个创作者的固定措辞；学习的是“具体事件先于观点”的内容方法。

## 硬失败

出现任意一项即退回修复：

- 新增素材没有提供的数字、人物、产品、结果或因果关系；
- 伪造第一人称经历；
- 把“可能、怀疑、未必”改成确定事实；
- 开头直接宣布“真正重要的是”“本质上”“说到底”等普遍结论；
- 用两句对仗判断代替观察，例如“A 决定第一次，B 决定第二次”；
- 连续命令读者“你应该、你必须、你一定要”；
- 营销号腔、翻译腔、成功学总结或没有证据的权威口吻；
- 跑题、错语言或为了完整而扩写无关信息。

## 独立质检

质检按以下顺序进行：

1. 忠实度：事实、对象、确定性和因果关系是否都来自素材；
2. 证据：观点前是否出现了素材里的具体信号；
3. 自然度：是否像当事人在分享，而不是老师在给所有人上课；
4. 信息增量：每段是否提供了新的事实、关系或判断；
5. 发布价值：读者能否从中获得一个值得收藏、讨论或继续追问的东西。

正常流程只生成一篇初稿。初稿通过质检就直接交付；只有失败时才修复一次。用户不会看到内部候选或评分。

## 人工迭代

每轮在产品外生成 12—15 篇不同类型正文，由审核者给出：

- 整体分数；
- 最好与最差的编号；
- 共同问题，例如说教、太像 AI、缺少细节、节奏拖沓；
- 真实发布意愿。

下一轮只根据稳定出现的共同问题修改 Skill。单条偏好不直接升级为全局规则，避免少量历史数据把生成带入负循环。
