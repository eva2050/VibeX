# 中文 X Post Skill v1.2 设计说明

## 背景与验收

v1.1 的首轮 15 条人工审核结果为 30 分，真实发布意愿 0/15。失败不是少量禁词没有覆盖，而是整个生成形状仍然把普通素材加工成完整、平滑、带结论的微型演讲；同时测试素材缺少近期事实，Skill 只能把旧观点写得更像文章，无法产生话题新鲜度。

v1.2 的人工发布门槛固定为：下一轮 15 条近期真实信号中，至少 6 条愿意直接发布且综合分至少 70。未达到门槛时不得把 v1.2 视为完成验证。

## 核心变化

生成质量拆成两层：

1. **素材层**判断有没有事件、实测、数字、公开反馈、具体约束或可靠来源。话题新鲜度只能来自这一层，文风不能制造新鲜事实。
2. **表达层**根据素材已有信号决定成稿形状，优先保留现场、动作、限制和不完整感，不再把所有输入整理成“现象—分析—结论”。

素材薄或只有抽象观点时，Skill 应明确降级为短表达，不得用金句、比喻、步骤或宏大结论伪造深度。

## 语料设计

仓库保存 40 条公开中文 X 样本的研究记录：24 条正样本、16 条反样本，覆盖至少 8 位作者。每条只保存短摘要和结构标注，不复制完整正文。

每条记录包含：

- 原帖 URL、作者、抓取时间和可用的发布时间/相对时间；
- 抓取时的回复、转发、点赞、浏览和收藏快照；
- 话题、内容形状、来源类型；
- `positive` 或 `negative` 标签；
- 值得学习的信号，或禁止学习的失败特征。

播放量不能直接决定标签。标题党、纯广告、无来源新闻复述、情绪放大、抽象训话和互动诱饵即使高播放，也可以是反样本。任何运行时规则必须至少由 3 位不同作者的样本共同支持，不能复刻单一作者。

## 输入诊断

在现有内容类型之外增加 `signalType` 和 `sourceStrength`：

| signalType | 含义 | 推荐形状 |
| --- | --- | --- |
| `first_hand_result` | 自己做了什么并有结果 | 动作/结果开头，保留限制 |
| `first_hand_test` | 实测产品、模型或工作流 | 测试对象、观察、短板 |
| `sourced_update` | 有明确来源的近期事件 | 来源事实、增量、边界 |
| `public_feedback` | 对产品或公开内容的具体反馈 | 指向具体设计/行为，不升华 |
| `data_snapshot` | 有可核对数字 | 数字与对象开头，解释口径 |
| `concrete_observation` | 有对象和具体行为但无数据 | 直接描述现象与疑问 |
| `abstract_opinion` | 只有泛化观点 | 降级为短句，不伪装爆款 |
| `thin_input` | 信息不足 | 忠实保留，不扩写 |

`sourceStrength` 分为 `strong`、`medium`、`weak`。强素材至少有两种具体信号；中等素材至少有一种；抽象或提取失败为弱素材。

## 生成契约

成稿不再套 family recipe，而是按信号选择一个 pattern card：

- **实测记录**：做了什么 → 看到了什么 → 仍不确定什么；
- **产品反馈**：点名具体界面/动作 → 说明摩擦 → 到此停止；
- **数据快照**：数字和对象 → 口径/条件 → 有边界的解释；
- **来源拆解**：谁发布了什么 → 最有用的增量 → 限制或反证；
- **现场片段**：具体场景 → 当下反应或疑问，不强行总结；
- **短判断**：只保留一个判断，最多补一个理由。

禁止把 pattern card 写成固定三段式。输出应保留输入本来的长短：一个短观察可以只有一两句，长文只能由足够多的事实和过程支撑。

## 反模式

除 v1.1 已有的事实忠实度约束外，新增拦截：

- “这个时代、未来已来、真正的拐点、底层真相”等时代宣言；
- “王炸、炸裂、恐怖、必看、彻底说透、颠覆认知、改命、黄金窗口”等内容农场放大词；
- 输入没有步骤却输出“第一、第二、第三”；
- 连续使用“不是 A，而是 B”制造伪反差；
- 用“很多人/大多数人都……”虚构共识或树立稻草人；
- 结尾给读者布置行动、升华人生或追加万能互动问题；
- 丢掉素材里的具体信号，只留下更抽象的总结。

## 运行边界

- Studio 继续一次只交付一篇，不新增用户设置。
- v1.2 不在每次生成时实时抓取 X；语料由开发侧维护，避免登录、限流、延迟和隐私问题。
- 语料只提供结构证据和反模式，不作为当前主题来源，也不把第三方句子注入 prompt。
- Auto 仍保留但不作为安全默认路径；本次不删除 Auto。
- 这里的“训练”指语料、诊断、提示词和评审规则迭代，不修改模型权重。

## 验证

自动验证包括：

- 40 条语料的数量、标签、作者覆盖、URL 和快照字段；
- 规则证据至少跨 3 位作者；
- 信号类型和强度路由；
- 说教、内容农场、伪步骤、伪反差和具体信号丢失；
- 版本归因和 Studio/Auto 集成无回归；
- 15 条第二轮内容不与语料出现连续 12 个及以上相同中文字符。

自动测试只能证明边界正确，不能替代人工发布意愿。第二轮 15 条必须使用近期真实来源，并由用户评分后决定是否继续迭代。
