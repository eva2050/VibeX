# 中文 X Skill v1.2 人工审核第 2 轮

这轮不看机器分，只看一件事：你是否愿意直接发布。

审核后只需要回复三项：整体分、愿意发布的编号、共同问题。当前验收门槛是至少 6/15 愿意发布，且整体分不低于 70。

## 01

**来源：** https://x.com/op7418/status/2075535634375082364

**信号类型：** 产品观察

**素材摘要：** 小红书允许把 vibe coding 做出的网页以“小工具”形式挂在图文或视频下方，平台负责发布，但不包生成。

**正文：**

小红书新出的「小工具」挺有意思：vibe coding 做出来的网页，可以直接挂在图文或视频下面，别人不用离开小红书就能玩。

它没自己做生成器，只接发布。这个取舍很小，但可能比再造一个 AI Builder 实用。

## 05

**来源：** https://x.com/dotey/status/2076371173244588135

**信号类型：** 来源拆解

**素材摘要：** JetBrains 使用 SkillsBench 的 86 个任务进行约 240 次实验，测试“穴居人式表达”是否能明显节省 Agent Token。

**正文：**

「让 Agent 像穴居人说话，能省 65% Token」听起来很诱人。

JetBrains 用 SkillsBench 的 86 个任务跑了约 240 次，结果输出 Token 只少了 8.5%。Coding Agent 真正吃 Token 的地方在工具调用、系统提示、Skills 和 MCP，聊天废话只是很小一块。

要省钱，先砍无关上下文和返工。把句子写成电报，没省到大头。

**信号类型：** 来源更新

**素材摘要：** Anthropic 的 Agent 基础设施对谈提到，团队正在减少固定流程控制，让模型在目标与边界内自行使用工具。

**正文：**

Anthropic 7 月 10 日那场 Agent 基础设施对谈，有个变化值得记：团队正在少写「先 A 再 B」的流程控制，多给目标和边界，让模型自己调工具。

但 harness 变薄，不等于管理变少。多 Agent 之后，新工作变成谁提方案、谁挑错、谁在卡住时接管。控制代码少了，协作设计反而多了。

**正文：**

Codex 这块简单得多：结果能预览，也能直接进 VS Code。

都已经让 Agent 操作浏览器和文件了，最后一步还要人肉找文件，挺割裂的。
